Saltar a las direcciones

Tres direcciones para el design system de la colección de Agent Skills

El mismo artefacto de ejemplo —un plan de cambio del proyecto ficticio meridian (idempotencia de webhooks)— renderizado en tres sistemas visuales radicalmente distintos. Los tres cumplen el mismo quality bar: autocontenidos, legibles sin JavaScript, navegables por teclado, AA, reflow a 320 px, print e impresión monocroma, reduced motion y exports que cierran el loop.

Evalúa cada dirección de arriba a abajo: procedencia → título → resumen → evidencia → decisión → comparación → código → timeline → diagrama → estados → export → quiz. Después hay una matriz comparativa y una recomendación.

A · Acta B · Consola C · Expedición Matriz y recomendación
DIRECCIÓN A

«Acta» — el artefacto como documento de registro

Tesis

Un artefacto durable es un acta de ingeniería: papel primero, pantalla después. La procedencia abre el documento como un colofón; la evidencia se cita y se registra; las decisiones se numeran y se firman. Nada compite con el texto.

Principios

  • Print-first: la versión impresa es la de referencia.
  • Jerarquía por reglas tipográficas, no por cajas.
  • Evidencia citable: cada afirmación remite a un registro de fuentes.
  • Decisiones como registro numerado con estado.
  • Motion ≈ 0; la interacción es sobria y secundaria.

Tokens (muestra)

papel tinta tinte acción positivo aviso peligro

Radius 0–2 px · reglas 1 px y dobles · sombras: ninguna · motion 0–100 ms

Tipografía

Serif de texto (Charter/Iowan/Georgia) como voz de lectura, también en títulos. Mono para procedencia, datos, labels y código. Sans casi ausente (sólo controles).

Fortalezas

  • Lectura larga y revisión: la mejor de las tres.
  • Print y monocromo impecables por construcción.
  • Procedencia y auditabilidad como identidad, no como pie.
  • Envejece bien; cero dependencia de moda visual.

Debilidades

  • Los editores interactivos se sienten «de papel».
  • Menos affordance lúdica para explorar (concept-lab).
  • Densidad de datos media: tableros grandes piden más grid.

Dónde brilla

change-blueprint (spec y plan), what-just-happened, feature-xray, deepen-the-codebase, three-code-paths en modo lectura.

Riesgos

Puede leerse como «documento estático» y desincentivar la interacción; los instrumentos muy dinámicos (build-with-notes vivo, quiz) necesitan disciplina para no romper la sobriedad.

Qué la hace original

Ningún artefacto del upstream trata la procedencia como colofón notarial ni ofrece registro de fuentes y de decisiones numerado. El print-first como contrato (no como afterthought) es una postura propia.

§2Registro de decisiones

D-01

Clave de idempotencia: header X-Meridian-Delivery

UUID por evento, generado al encolar. El consumidor descarta entregas repetidas comparando la clave, no el cuerpo.

Estado: propuesta · Reversible: sí (el header es aditivo) · Cambiar si: los consumidores no pueden leer headers.

Alternativa considerada: hash SHA-256 del payload

Evita el header, pero colisiona con eventos legítimamente idénticos (dos cargos iguales el mismo día) y ata la clave al cuerpo serializado. Descartada por ambigüedad semántica.

Respuesta de una línea para cambiarla: «Usa hash del payload en lugar del header.»

D-02

Presupuesto de reintentos: 6 intentos / 24 h con backoff exponencial + jitter

Al agotarse, el evento pasa a la cola muerta con causa. Sin presupuesto, un endpoint caído multiplica tráfico ×40 [E2].

Estado: propuesta · Owner: pendiente de asignar en aprobación.

§3Comparación: dónde vive la cola

OPCIÓN 1 · PROPUESTA

Outbox en Postgres

  • Transaccional con el negocio: cero eventos fantasma.
  • Sin infraestructura nueva; opera el mismo backup.
  • ~500 eventos/s medidos en staging; +8 ms p50.

Elegirla si el volumen se mantiene < 1.000 ev/s (hoy: 120 ev/s).

OPCIÓN 2

Redis Streams

  • ~10.000 ev/s; consumidores en grupo nativos.
  • Infra nueva + reconciliación con Postgres al fallar.
  • Entrega at-least-once que exige la misma idempotencia.

Elegirla si el volumen se multiplica ×10 este año (roadmap: no).

§4Código con más riesgo

El registro del intento debe confirmarse antes del POST; si se invierte el orden, reaparece el duplicado.

- const res = await post(endpoint, event, { timeout: 10_000 });
+ const attempt = await attempts.begin(event.id, delivery.key);  // D-01
+ const res = await post(endpoint, event, {
+   timeout: 10_000,
+   headers: { "X-Meridian-Delivery": delivery.key },
+ });
+ await attempt.close(res.status);   // presupuesto D-02 se evalúa aquí

DIRECCIÓN B

«Consola» — el artefacto como instrumento de banco

Tesis

Un artefacto de trabajo es un instrumento: estado siempre visible, cada dato con timestamp y traza, controles francos y operables con teclado. La página es un banco de pruebas, no una publicación.

Principios

  • Estado primero: barra de estado y gates con estado grande.
  • Densidad alta; cada panel se gana su espacio.
  • Teclado como ciudadano de primera (atajos visibles).
  • Evidencia como log trazable, no como prosa.
  • Feedback instantáneo; cero motion decorativa.

Tokens (muestra)

base panel cabecera acción positivo aviso peligro

Radius 0 px · paneles con header propio · grid de 20 px visible en diagramas · motion 0–100 ms

Tipografía

Mono estructural (headers de panel, labels, datos, timeline) + sans neutral para prosa en «islas» de medida limitada. Sin serif.

Fortalezas

  • Vistas vivas y diagnóstico: estado y progreso inequívocos.
  • Editores y quizzes con affordance clara.
  • Trazabilidad: timestamps + rutas en todo.
  • Densidad útil en pantallas grandes.

Debilidades

  • Lectura larga más dura; exige islas de prosa disciplinadas.
  • Puede leerse como «IDE genérico» si se descuida.
  • El gris técnico pide cuidado para no volverse frío.

Dónde brilla

build-with-notes (vista viva), find-the-cause, do-i-understand-this, what-just-happened técnico, editores de feel-the-flow, draw-the-flow de sistemas.

Riesgos

Sobre-instrumentar documentos que sólo quieren ser leídos (spec de change-blueprint); la densidad castiga en móvil si no se colapsa el rail y las tablas.

Qué la hace original

Gates de decisión como estados de instrumento, navegación con atajos visibles, evidencia como log con timestamp y papel milimetrado bajo los diagramas: nada de esto existe en el upstream, que es editorial en todos sus artefactos.

DIRECCIÓN C

«Expedición» — el artefacto como ruta con mapa

Tesis

El lector es un viajero: primero el mapa completo, luego la ruta por estaciones numeradas. Cada hallazgo es un espécimen etiquetado; cada decisión, una bifurcación señalizada. La orientación nunca se pierde.

Principios

  • Mapa antes que ruta: overview obligatorio y clicable.
  • Waypoints numerados conectados por una ruta visible.
  • Evidencia como ficha de espécimen (origen + confianza).
  • Bifurcaciones: la ruta elegida en color pleno, la descartada punteada.
  • Targets grandes; motion suave y prescindible.

Tokens (muestra)

bruma ficha tinta ruta positivo aviso peligro

Radius 12–16 px · píldoras · rutas punteadas · motion 150–250 ms ease-out (retirable)

Tipografía

Sans humanista (Seravek/Segoe UI) para display y cuerpo; mono para coordenadas, orígenes y datos. Sin serif. Pesos fuertes y tracking apretado en títulos.

Fortalezas

  • Orientación: imposible perderse en documentos largos.
  • Pedagogía: explainers y quizzes se sienten naturales.
  • Fact/inference como sistema visual (espécimen), no como nota.
  • Táctil: móvil de primera, no adaptación.

Debilidades

  • Menor densidad: tablas y logs grandes piden más espacio.
  • El tono guía puede sobrar en informes de incidente serios.
  • Más superficie de card = más ruido en artefactos muy cortos.

Dónde brilla

concept-lab, learning-workbench, draw-the-flow, interface-directions, feel-the-flow, do-i-understand-this, three-code-paths exploratorio.

Riesgos

Riesgo de «infantilizar» artefactos de auditoría (what-just-happened) si la metáfora no se atenúa; exige disciplina para que el wayfinding no se coma el contenido.

Qué la hace original

Operacionaliza overview→detail y hecho/inferencia como geometría (mapa, waypoints, especímenes, bifurcaciones), no como convención editorial. No existe nada cartográfico en el upstream.
